# ubuntu.arr — Home Server Stack

Modular Docker Compose home server: an **infrastructure** stack (reverse proxy, auth, VPN, DNS, monitoring) plus an **arr-stack** (VPN-protected torrenting, *arr automation, Jellyfin streaming). Both stacks share a common Docker network so Caddy can reverse-proxy across them.

---

## Stacks

### Infrastructure (`infrastructure/`)

| Service | Image | Purpose |
|---|---|---|
| **Caddy**       | `caddy:2-alpine`            | Reverse proxy with automatic HTTPS (self-signed by default) |
| **Authelia**    | `authelia/authelia:latest`  | Single-sign-on portal for protected admin UIs |
| **Portainer**   | `portainer/portainer-ce`    | Docker management UI |
| **DDNS Updater**| `qmcgaw/ddns-updater`       | Dynamic DNS client (Cloudflare, Namecheap, …) |
| **wg-easy**     | `ghcr.io/wg-easy/wg-easy`   | WireGuard server with web UI |
| **Dashdot**     | `mauricenino/dashdot`       | System-resource dashboard (embedded in Homarr) |

Creates the shared `home_network`. Must be deployed first.

### Arr-stack (`arr-stack/`)

| Service | Image | Purpose |
|---|---|---|
| **Gluetun + Deluge** | `qmcgaw/gluetun` + `linuxserver/deluge` | VPN-protected BitTorrent (Mullvad / NordVPN / 70+ providers) |
| **Prowlarr**    | `linuxserver/prowlarr`     | Indexer manager |
| **Radarr**      | `linuxserver/radarr`       | Movie automation |
| **Sonarr**      | `linuxserver/sonarr`       | TV automation |
| **Lidarr**      | `linuxserver/lidarr`       | Music automation |
| **Bazarr**      | `linuxserver/bazarr`       | Subtitle automation |
| **Jellyfin**    | `linuxserver/jellyfin`     | Media streaming |
| **Jellyseerr**  | `fallenbagel/jellyseerr`   | Media request portal |
| **Homarr**      | `ghcr.io/homarr-labs/homarr` | Dashboard |

## Quick start

```bash
git clone https://github.com/stivce/ubuntu.arr.git
cd ubuntu.arr

cp .env.example .env
# Edit .env — at minimum:
#   DOMAIN=
#   HOMARR_SECRET_KEY=               (openssl rand -hex 32)
#   AUTHELIA_DEFAULT_PASSWORD_HASH=  (see Authelia section below)
#   VPN_SERVICE_PROVIDER=            (mullvad | nordvpn) and matching credentials

./arr.sh deploy
```

`arr.sh` is the single entrypoint — run it with no arguments to see all commands. `deploy` is idempotent: it generates Authelia secrets if missing, copies the Caddyfile only if not yet deployed (set `FORCE_CADDYFILE=true` to overwrite), and refuses to start if `HOMARR_SECRET_KEY` is empty.

```bash
./arr.sh deploy           # deploy full stack (infrastructure + media)
./arr.sh destroy          # stop containers, keep volumes + configs
./arr.sh destroy --volumes  # also remove docker volumes
./arr.sh destroy --all    # also delete config dirs on disk (irreversible)
./arr.sh backup           # back up all service configs
./arr.sh restore <file>   # restore from a backup archive
./arr.sh status           # show container status for both stacks
./arr.sh logs [service]   # follow logs
```

## Service URLs

All services are reachable through Caddy at `https://<service>.${DOMAIN}` once you've installed the Caddy root CA on your devices (download from `https://ca.${DOMAIN}/root.crt` after first deploy).

| Service | URL | Direct (LAN) | Behind Authelia |
|---|---|---|---|
| Authelia portal | `auth.${DOMAIN}`        | —                       | self |
| Portainer       | `portainer.${DOMAIN}`   | `:9000` / `:9443`       | no (own auth) |
| DDNS Updater    | `ddns.${DOMAIN}`        | `:8080`                 | yes |
| WireGuard UI    | `wireguard.${DOMAIN}`   | `:51821`                | yes |
| Dashdot         | `dash.${DOMAIN}`        | —                       | no (iframe-friendly) |
| Homarr          | `homarr.${DOMAIN}`      | `:7575`                 | no |
| Prowlarr        | `prowlarr.${DOMAIN}`    | `:9696`                 | no |
| Radarr / Sonarr / Lidarr / Bazarr | `<arr>.${DOMAIN}` | `:7878` / `:8989` / `:8686` / `:6767` | no |
| Jellyfin        | `jellyfin.${DOMAIN}`    | `:8096`                 | no |
| Jellyseerr      | `jellyseerr.${DOMAIN}`  | `:5055`                 | no |
| Deluge          | `deluge.${DOMAIN}`      | `:8112`                 | no |

WireGuard's public UDP listener stays on `51820/udp` — forward that on your router.

## Authelia

Authelia protects the WireGuard, DDNS, and CA-download web UIs. Generate an argon2 password hash for the admin user and put it in `.env`:

```bash
docker run --rm authelia/authelia:latest \
    authelia crypto hash generate argon2 --password 'YOUR_PASSWORD'
# → put the resulting hash into AUTHELIA_DEFAULT_PASSWORD_HASH
```

`AUTHELIA_DEFAULT_USER` defaults to `admin`. `AUTHELIA_JWT_SECRET`, `AUTHELIA_SESSION_SECRET`, `AUTHELIA_STORAGE_ENCRYPTION_KEY` are auto-generated by `deploy.sh` if blank.

## HTTPS

The Caddyfile uses `local_certs` (Caddy's internal CA) by default — works without any DNS or internet exposure, but browsers won't trust the cert until you install the root CA from `https://ca.${DOMAIN}/root.crt`:

```bash
# macOS
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain caddy-root-ca.crt
# Linux
sudo cp caddy-root-ca.crt /usr/local/share/ca-certificates/ && sudo update-ca-certificates
# Windows: double-click → Install → Local Machine → Trusted Root CAs
```

To switch to Let's Encrypt: replace `local_certs` in `infrastructure/Caddyfile` with `email {$ACME_EMAIL}`, set `ACME_EMAIL` in `.env`, and re-run `FORCE_CADDYFILE=true ./deploy.sh`.

├── arr.sh                    # single entrypoint (deploy, destroy, backup, restore, status, logs)
├── deploy.sh / destroy.sh    # underlying orchestrators called by arr.sh
├── lib/deploy-helpers.sh     # shared bash helpers (env, secrets, ddns, caddy)
├── infrastructure/
│   ├── docker-compose.yml
│   ├── Caddyfile             # template — copied to runtime config on first deploy
│   ├── deploy.sh
│   └── authelia/
│       ├── configuration.yml # template (rendered at deploy time)
│       └── users_database.yml
└── arr-stack/
    ├── docker-compose.yml
    ├── deploy.sh
    ├── backup.sh             # API + tar-based backups (called by arr.sh backup)

Runtime data lives **outside** the repo:

```
/data/
├── docker/
│   ├── infrastructure/config/{caddy, portainer, ddns-updater, wg-easy, authelia}/
│   └── arr-stack/config/{gluetun, deluge, prowlarr, radarr, sonarr, lidarr, bazarr, jellyfin, jellyseerr, homarr}/
└── media/{movies, tvseries, anime, music, iptv, downloads}/
```

## Backup & restore

```bash
./arr.sh backup                # writes to /data/backups/arr-stack/, keeps last 5
./arr.sh restore /data/backups/arr-stack/arr-stack-backup-YYYYMMDD-HHMMSS.tar.gz
```

`backup.sh` uses each *arr app's API for hot DB backups (Sonarr/Radarr/Lidarr/Prowlarr) and tars the rest (Bazarr/Jellyfin/Jellyseerr/Homarr/Deluge). Caddy PKI, Authelia users, and WireGuard peers are included from the infrastructure side. Media files are **not** backed up — sync `/data/media/` separately with `rsync` or `rclone`.

`restore.sh` snapshots the current configs to `/data/docker/restore-snapshots/` (last 3 kept) before applying, and patches Deluge's `web.conf` so the UI reconnects after restore.

## Adding a new service

1. Append the service block to `infrastructure/docker-compose.yml` (or `arr-stack/docker-compose.yml`):
    ```yaml
    newservice:
      image: …
      container_name: newservice
      networks: [home_network]
      volumes:
        - ${INFRASTRUCTURE_CONFIG_DIR:-/data/docker/infrastructure/config}/newservice:/config
    ```
2. Append a route to `infrastructure/Caddyfile`:
    ```caddyfile
    newservice.{$DOMAIN} {
        reverse_proxy newservice:PORT
    }
    ```
3. Add the runtime config dir to the `mkdir -p` line in the deploy script.
4. Re-run `FORCE_CADDYFILE=true ./deploy.sh` so the updated Caddyfile is copied across, then `docker exec caddy caddy reload --config /etc/caddy/Caddyfile`.

## Troubleshooting

```bash
# VPN actually working?
docker exec gluetun wget -qO- https://api.ipify.org   # should differ from your home IP
curl -s https://api.ipify.org

# Caddy can't reach an arr service?
docker exec caddy nc -zv radarr 7878
docker network inspect home_network | grep radarr

# arr can't reach Deluge?
docker exec radarr nc -zv deluge 58846   # the alias resolves to gluetun's network
```

PUID/PGID in `.env` should match the host user that owns `/data/`. Permission errors mean you're set to a UID that doesn't own the bind-mounted path.
